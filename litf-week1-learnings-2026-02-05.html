The Core Realization

The hard part isn't getting them to the frontier.

It's keeping them there when they naturally want to build the obvious thing.

"CAPY Card has an MVP competing with DOT/Linq on features. Without intervention, they'd build 'digital business card but with AI' — the classic incremental trap."

Product-first teams drift incremental. Curiosity-first teams find LITF faster.

3/6

What Surprised Me

The Inversion Framework worked immediately

Jai & Satith ran the four questions. Result: they recognized 3D content creation is commoditizing fast.

They got frustrated: "What should we do?"

Key insight

Frustration is a positive signal. "What should we do?" means they're actually thinking instead of building on quicksand.

Samuel (Hackathon Team) landed on autonomous agents without any pushing. Curiosity-first works.

4/6

Patterns Across Teams

  • Less technical ≠ less frontier
    Think Tank can't "build an agent" but can run experiments and document friction.
  • Domain interest is fine if capability edge is real
    Sports coaching focus → video analysis frontier. The domain doesn't have to be frontier—the capability exploration does.
  • Each team needs a different experiment
    Technical teams build. Non-technical teams probe and document where AI breaks.
5/6

Week 2 Plan

Four experiments, same goal

Jai & Satith: Explore Sequel, re:motion. Find the durable layer after content creation commoditizes.
CAPY Card: Push from typed input → voice capture. Record mock conversations, test with recruiters.
Hackathon: Install OpenClaw, set up ONE autonomous task, run 3 days, document friction.
Think Tank: Feed 3 soccer clips to GPT-4V, ask for coaching analysis, document what breaks.
