Where I Started

The Plan: Run It From Behind the Scenes

Friday deadline: deploy bootcamp agents for 80 students.

My original role: team support. Build it, hand it off, watch from the sidelines while Michael and Laura run the session.

The architecture I'd designed: momentum coaching across all bootcamp stages. Felt solid.

Then I had a thought...

2/5

The Pivot

What If I Actually Used It?

Instead of running the deployment, I decided to attend as a participant.

Ultimate dogfooding moment: use my own bootcamp agent alongside 65 other students.

This forced two things:

  • Handoff materials had to be airtight for Michael & Laura
  • I'd experience the agent the way students actually would
3/5

The Architectural Shift

Interview Recall, Not Personal Experience

Critical realization mid-build:

"Students interviewed someone else's problem. The agent needed interview-recall scaffolding, not personal experience prompts."

Core value shifted to solution ideation using pre/post consumption framework (learned from Craig's friction agent).

Final architecture: two-tier approach (strategic frameworks always, concrete product ideas on request) + individual memory blocks with team tags.

4/5
