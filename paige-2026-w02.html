3/7

The Pivot

The onboarding sequence isn't for students at all

It's for mentor pre-training.

Craig/Cameron/Erica fill in context ON BEHALF of founders, then the agent feels "mysteriously knowing" from the first interaction.

The Key Insight

That shift is where trust is built.

Completely different design problem. Same architecture, new purpose.

4/7

What I Built

Two-phase system

Phase 1: Pre-Training Mode

Manager/mentor interviews agent ABOUT the founder. 3rd person questions, memory block population, gap tracking.

Phase 2: Founder Mode

Agent demonstrates knowledge immediately. Uses tick-tock onboarding to fill remaining gaps — but founder never sees the checklist. Feels conversational, not transactional.

5/7

The Catch

The first draft opening

"Hey [Name]! Here's what's about to happen: I'm going to show you what I already know about you..."

It explained the process clearly. But if the first message feels like "setup overhead," you've already triggered the friction you're trying to avoid.

The opening needs to do double duty: ORIENT (what's about to happen) AND ENERGIZE (why this is worth doing).

6/7

The Fix

Same structure. Different energy.

"Hey [Name]! I'm already up to speed on you and [Business Name], so we're not starting from zero. Let me show you what I know — you tell me what I'm missing — then [X] quick questions and we can jump into whatever you need. Whole thing takes 2-3 minutes. Ready?"

Momentum instead of overhead.
