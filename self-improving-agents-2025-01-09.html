The Setup

The Problem

We had a working agent architecture: core memory blocks for state, archival for long-term storage, skills that load/unload based on context.

But improvement was manual. When a skill needed upgrading, a human had to notice, analyze, and rewrite it.

The question: Can the agent drive its own improvement cycles?

3/8
The Insight

Self-Prompting

The Kavindi analysis worked because it was example-driven, not theory-driven.

The unlock: what if the agent could do this to itself?

Instead of waiting for instructions, the agent generates its own improvement prompt, then executes against it.

"Analyze [example] and extract patterns for [target]"
"What would a 10x better version of [target] look like?"
"What's missing from [block] based on recent usage?"

The agent has the context. It just needed permission and structure to use it.

Open Question

Skills live in archival. When growth_boost improves a skill, it creates a new version. But what if multiple agents use the same skill?
