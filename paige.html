Starting Point

The onboarding problem persists

Same core challenge from December: students showed apprehension toward using agents.

The December breakthrough was "let the agent read documents itself" instead of building extraction pipelines. That simplified the architecture.

But the question remained: how do you build trust in the first interaction? How do you make onboarding feel like a conversation, not a form?

1/6

The Trade-off

Structured vs. Conversational

Two modes kept emerging:

Structured Mode
Predictable, systematic, good for collecting info. But feels like a checklist. Students tune out.
Conversational Mode
Natural, responsive, builds rapport. But can meander. Might miss critical context.

The question: pick one, or find a way to blend them?

2/6

The Decision

Hybrid Default with Adaptive Mode-Switching

Option A — Selected
Start structured (predictable, trust-building), but switch to conversational ("tick-tock") when signals detected.

Key design choices:

  • Conversational trust checkpoint (not checklist-y)
  • ONE synthesis observation (proves intelligence)
  • Shorter question rationale (one sentence max)
  • Pre-loaded ZIP Launchpad ecosystem data
  • No mention of "gaps" — feels judgmental
