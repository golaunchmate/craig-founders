Weekly Update

Paige's Week

From Setup Friction to Agent Autonomy

December 2025 // scroll to read

Starting Point

The core problem hasn't changed

Came into the week with the same thing on my mind: students showed apprehension toward using Launchmate last semester.

The goal is still the same—figure out how to make agents genuinely useful so students WANT to use them, not because mentors are forcing it.

1/6

Early Week

Nailing the onboarding

Dove into designing the onboarding sequence. Built a 5-question framework mapped to memory blocks (founder_profile, business_context, ecosystem_map).

Added "trust checkpoints" after every 3-4 pieces of info—show students what the agent captured, prove it's retaining context, address the skepticism problem directly.

Then extended beyond onboarding: created 10 "crisis-mode capabilities" (Unblock Me, Gut Check, Prepare Me, etc.) with detection logic so agents can contextually suggest help when needed.

2/6

Mid-Week Shift

Setup friction became the blocker

Realized the onboarding questions, while well-designed, still felt like WORK. Students are busy, backgrounds vary—if building up context feels like a burden, they won't finish setup at all.

Classic trap: need context to be useful (address skepticism), but gathering context creates friction (causes drop-off).

Started exploring pre-loading: what if agents could ingest info students already provided elsewhere? Applications, intake transcripts, analysis docs.

3/6

The Problem

Then I hit reality

Designed a JSON schema for what pre-load data should look like. Then realized: no structured database exists.

All the source material is unstructured—transcripts, documents, mixed formats. So I started thinking about extraction pipelines, staff pre-processing documents...

It was getting complicated fast.

4/6

Late Week Breakthrough

Let the agent do the work

Had an "obvious in hindsight" realization: why build extraction pipelines? The agent can just... read the documents itself.

Instead of:

Documents → extraction tool → structured data → agent → student confirms

Just do:

Documents → agent reads & organizes → student confirms

Agent receives PDFs, transcripts, pitch decks on initialization. It decides what's relevant. First interaction shows pre-loaded context with 3-4 targeted questions to fill gaps. Total student time: 3-5 minutes instead of 15+.

5/6

Where I Am Now

Simpler is stronger

Architecture shifted from "how do we structure data FOR the agent" to "let the agent structure data ITSELF."

More elegant, scales better, removes friction at every level. Same process works for 5 students or 500.

Now exploring whether Letta supports document processing on initialization, and designing what that first "trust checkpoint" interaction looks like.

  • Removed staff bottleneck
  • Removed extraction complexity
  • Agent owns its context-building

Feels like a breakthrough moment—the kind where you realize you were overcomplicating it and the simpler approach is actually stronger.
