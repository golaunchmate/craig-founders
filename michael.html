Learning

What this revealed

Growth baseline
~2 weeks of daily use before real value kicks in
Better metric
Tool call density may be better signal of agent growth than message count
The design problem
Each user has unique problem space — how to grow agent that pushes back on right things?

Catch-all instructions ("be skeptical") don't lead to good behavior. Need specificity.

5/7

Key Discovery

"Grow Quicker For Me"

The agent self-identified its own mistakes:

  • Overcorrection
  • Assumption without verification
  • Overcomplication
  • Absolutism

Then it asked: "Would you like me to encode these as guardrails for myself?"

Potential feature: agent auto-identifies its own error patterns and asks permission to self-correct. Users see the agent learning FROM them.
