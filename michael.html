Michael's Week

From Friction Analysis to Skill Architecture

December 15–19, 2025 // scroll to read

Starting Point

What makes LaunchMate different?

Started the week focused on foundations—defining what makes LaunchMate different from generic AI.

Built a friction analysis framework covering 16 specific frictions founders face with AI tools.

Landed on the core insight: trust + meaning are the locks. Founders don't believe AI can truly know them, and even when it helps, they can't see the value.

1/5

Early Week

Crisis mode capabilities

That insight led to designing 10 crisis mode capabilities—patterns for founders in high-stakes moments:

UNBLOCK_ME GUT_CHECK PREPARE_ME REALITY_CHECK PIVOT_OR_PERSIST + 5 more

Design principle: < 60 seconds to value.

If the agent can't help fast when it matters, the trust never builds.

2/5

Midweek Shift

Restructuring the validation playbook

Shifted focus to the validation playbook—the structure for guiding In-Flight founders through the whole flow: Neal Test → prototype → presales → delivery.

Key realization: the Neal Test isn't just customer discovery. It's the learning phase that informs what to build.

Had to restructure the whole framework around that insight. The Neal Test comes first because everything else depends on what you learn there.

3/5

Late Week

Making the agent context-aware

Then explored skill activation—how does the agent know which mode to be in?

Landed on a hybrid approach: store skills in archival memory, swap them into an active_skill block based on phase.

Built and tested two skills:

neal_test_coach presales_closer

Swap works. Agent can shift personalities based on where the founder is in their journey.

4/5

Where I Am Now

Architecture is solid

  • Friction framework: done (16 patterns)
  • Crisis mode capabilities: designed
  • Validation playbook: restructured
  • Skill swap system: working

Framework and registry pattern established. The pieces are in place.

Next up: testing with real founder interactions.
