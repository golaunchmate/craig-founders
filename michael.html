The Experiment

Built my own agent from scratch

Back from two weeks off. Instead of jumping into LaunchMate's founder agent, I did something different.

Built a chronic condition management agent for myself. Used it multiple times daily for two weeks straight—logging symptoms, meals, tracking variables.

Technical setup: 90k context window, precise daily_log memory block, week index + food index for archival lookups. Installed Letta as PWA on my phone.

Wanted to understand how agents actually grow by living it myself.

Where It Failed

Specific failure modes

  • Didn't know me immediately Gave recommendations that didn't track with long-term patterns I already knew. Treated me as blank slate.
  • Jumped to broad estimates Vague nutritional impact. Had to push: "give me estimates on EACH ITEM in this meal."
  • Variables grew unmanaged Symptom list expanded organically. Agent didn't proactively organize—I had to direct that.
  • Required constant babysitting Had to say "log it" or watch tool calls to ensure it actually persisted information.
3/8

The Hard Truth

Average users won't do this

I'm AI-literate. I know when to challenge the agent, when to ask for specificity, when to watch tool calls.

Normal users won't do any of that.

They'll accept broad estimates. Miss when the agent doesn't log something. Won't press into new expert info the agent surfaces.

What doesn't work
Catch-all instructions like "be skeptical" or "retrace your output steps" don't lead to good behavior. Too vague. The agent needs SPECIFIC patterns to push back on.
4/8

The Discovery

Agents can self-identify mistakes

Asked the agent: "What are the ways I've had to push back on you?"

It identified 4 error patterns with specific examples:

  • Overcorrection — swinging too far in response to feedback
  • Assumption without verification — acting on inferred context without checking
  • Overcomplication — adding unnecessary complexity
  • Absolutism — speaking definitively when uncertain

Then it proactively asked: "Would you like me to encode these as guardrails for myself?"

5/8

Product Insight

"Grow Quicker For Me"

Feature concept
The agent auto-identifies its own failure patterns and asks permission to self-correct. Users see the agent learning FROM them, not just FOR them.

The agent that works is one that grows WITH the user—learning their specific failure modes. Not static instructions, but an evolving model of "what I tend to get wrong with YOU."

Possible approaches:

• Agent self-identifies errors and encodes guardrails
• Domain-specific push-back patterns as skills
• Track user corrections → build "things I get wrong" profile
• Proactive verification: ask "did I get this right?"

Architecture

Skill loading works

Built and tested a skill swap system:

neal_test_coach
Pushes for manual delivery, surfaces patterns, resists premature building
presales_closer
Tracks pipeline obsessively, handles objections, creates urgency
weekly_reflection
Structured look-back/look-forward with "pass and catch" method

Test result: Skill swap works. Agent behavior changes based on loaded skill. Created activation framework with priority rules.
