1/6

Discovery

Agent growth happens through TWO channels

Shared Memory Blocks
Master updates a block → all clones see it instantly. Knowledge push.
A2A Async Messaging
Clone sends message to master → arrives as system message → becomes part of lived experience.
Clones learn → A2A to master → master distills → updates shared blocks → ALL clones benefit

Collective intelligence through bidirectional growth.

2/6

The Shift

The "magic" isn't in the config.

The realization

An agent's message history IS its learned behavior — tone, tool call patterns, edge case handling, compacted summaries.

We thought cloning meant copying configuration and memory blocks.

But the lived experience — the actual conversations — is where behavior emerges.

This changed the entire cloning strategy.

3/6

Key Decisions

What we built and why

Export/Import deep clone only
Templates = clean slate (no history). Conversations = can't seed. Deep clone carries everything.
8 focused tool primitives
Universal API tool was fragile (datetime issues, param collisions). Focused tools are stable.
Optional target_agent_id
Default = self (via env var). Explicit ID = target child. Clean self-vs-other pattern.
4/6

The Primitives

Self-orchestrating through client injection

Every tool gets a pre-initialized client with full API access. The FLBL agent can autonomously:

clone_agent send_agent_message create_shared_block update_block list_my_blocks list_agents manage_agent_tools create_tool

The agent bootstrapped itself — used its own tools to attach its own tools via client injection.
