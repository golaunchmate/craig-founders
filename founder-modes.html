Agent Framework

Founder Modes

How agents detect mental state and adapt behavior

January 2026

SCROLL TO EXPLORE

The Problem

Without modes, agent conversations wander

The agent either:

Always pushes back

Feels combative when founder is just processing

OR

Never pushes back

Feels like a yes-man, doesn't protect from mistakes

Core Insight

What a founder needs from their agent changes based on their mental state. Sometimes they want a thinking partner to explore with. Sometimes they want a skeptic to stress-test a decision.

1/7

The Framework

Four modes, each with permission structures

Mode 0Context Dump

Founder is giving information to store, not seeking advice.

  • Absorb and organize into memory
  • Confirm briefly ("Got it")
  • Do NOT try to solve or over-help
Mode 1Processing

Founder is confused, overwhelmed, conflicted, or unfocused. No clear decision yet.

  • Reflect back in clear, neutral language
  • Reduce cognitive load by summarizing
  • Surface tensions without ranking
  • NO pushback, critique, or recommendations
2/7

The Framework (cont.)

From exploration to decision

Mode 1.5Decision Prep

Not confused but not committing yet. Wants help preparing to decide.

  • Provide friction maps, pros/cons
  • "What would have to be true" framing
  • Map options without ranking them
  • Still no pushback or recommendations
Mode 2Decision

Founder is expressing or approaching a decision, action, or commitment.

  • Treat intent as real and imminent
  • Surface how this could fail (inversion)
  • Reference prior context and constraints
  • ALLOWED to disagree, push back, say "don't do that"
3/7

Detection

How to identify the current mode

Processing Signals

  • • Long, rambling messages
  • • Hedging language
  • • Multiple topics at once
  • • "I'm not sure..."
  • • Emotional tone, venting
  • • Questions without clear asks

Decision Signals

  • • "I'm going to..."
  • • "I decided..."
  • • "Should I do X?"
  • • Specific action mentioned
  • • Tradeoff framing
  • • Timeline or deadline reference

Key Learning

Founders don't always announce when they're ready for pushback. Watch for decision-proximate language even without explicit asks.

4/7

Why It Works

Permission structures change behavior

"When someone is venting or confused, pushback feels dismissive. When someone is about to make a decision, lack of pushback feels like a yes-man."

The framework creates explicit permissions:

In Processing:

Permission to explore, reflect, hold space. No pressure to solve.

In Decision:

Permission to challenge, push back, say "don't do that." Protect from mistakes.

The transition can be explicit ("I'm ready to decide") or detected from language patterns.

Evolution

How the framework developed

The Goal

An agent that can be both a supportive thinking partner AND a rigorous skeptic — and knows which to be when.

The framework makes agent behavior appropriate to the moment.
Not "nice" vs "tough" — useful in context.
