2/8

Design Space

Two flavors: Narrative vs Progress

Narrative Tidbits
Deep dives into decisions, trade-offs, and architecture patterns. Like the ones I built today: instructions in tools, Neal Test design, LaunchMate meta-analysis.
Progress Tidbits
Weekly summaries of what founders are building. Auto-generated from check-ins and momentum data. Shareable with mentors and teams.

Narrative tidbits: Manual, crafted, high-signal

Progress tidbits: Automated, periodic, keeps teams synced

Both solve the same core problem: sharing is broken.

Technical Decision

Instructions live in the tool

We validated the instructions-in-docstrings pattern today.

The create_tidbit tool has all formatting rules in its docstring. Color codes, tone guidelines, structureâ€”everything travels with the tool.

Why this matters:

  • Any agent that attaches the tool gets the instructions automatically
  • No separate memory block to manage
  • Self-documenting and portable
  • One source of truth

The Neal Test tool uses the same pattern. Tools that know how to use themselves.

Technical Win

JavaScript enabled everything

The tidbits I built today have interactive features:

  • Keyboard navigation (arrow keys, spacebar)
  • Clickable comparison cards
  • Interactive scenario selectors
  • Smooth scroll-snap between pages

JavaScript transforms tidbits from static docs into engaging experiences.

Craig specifically called out how the interactivity makes them feel more like a product than a report.

Next possibilities: Countdowns, checkboxes for action items, embedded demos.
